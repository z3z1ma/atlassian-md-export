# Implement Attachments, Indexes, Clean, Verify, And CLI Orchestration

## Scope

Implement attachment metadata/download behavior, filename safety, deterministic indexes, manifest updates, local verification, destructive cleanup semantics, and end-to-end CLI orchestration for Jira export commands using the client/state/rendering components from prior tickets.

## Acceptance Criteria

- Attachment behavior is metadata-only by default.
- `--download-attachments`, `--attachment-max-mb`, and repeatable `--attachment-include` are implemented.
- Downloaded attachments are stored under `attachments/ISSUE_KEY/ATTACHMENT_ID-safe_filename`.
- Attachment writes are atomic and skipped downloads remain visible as metadata.
- Filename safety prevents traversal and unstable/invalid local filenames.
- Indexes generated:
  - `indexes/by-status.md`
  - `indexes/by-assignee.md`
  - `indexes/by-epic.md`
  - `indexes/stale.md`
  - `indexes/all.md` if implementation keeps it simple
- Index links are relative and deterministic.
- `manifest.json` is deterministic and contains non-secret export metadata and hashes.
- `verify --out DIR` validates manifest, state, issue files, hashes, index links, and downloaded attachment references without contacting Jira.
- `clean --out DIR --remove-missing` deletes issue files and downloaded attachment directories absent from the last successful representative pull, preserves SQLite history, and fails without deletion if no representative run exists.
- `pull`, `issue`, `comments`, `attachments`, `index`, `verify`, and `clean` are wired to real implementation paths rather than scaffold placeholders.
- `pull --project`, `pull --jql`, and `pull --issue` fetch Jira data, fetch comments, write raw JSON and Markdown atomically, update state, update manifest, and regenerate indexes.
- `issue KEY... --out DIR` performs exact repulls using key-constrained JQL.
- `comments KEY... --out DIR --force` refreshes authoritative comments and rerenders existing local issues where raw issue JSON exists.
